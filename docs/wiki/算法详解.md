# 🔐 算法详解

## 设计理念

传统加密算法对每个字节使用同一种运算（如简单的加法或异或），攻击者如果知道了一部分明文和对应密文，就可以反推出密钥，进而破解所有数据。

NextXi v3.0 的核心创新：**每个字节使用不同算法**，循环轮转 4 种加密操作。

## 四种加密算法

### 1. Add —— 凯撒加法

```
密文 = (明文 + 密钥字节) mod 256
```

最经典的加密方式，凯撒密码的字节版。

**加密：** `c = (p + k) & 0xFF`
**解密：** `p = (c - k) & 0xFF`

### 2. XOR —— 异或加密

```
密文 = 明文 XOR 密钥字节
```

异或是**自逆运算**，加密和解密用同一个操作。

**加密：** `c = p ^ k`
**解密：** `p = c ^ k`（和加密一样！）

### 3. Rot —— 位旋转

```
密文 = 明文循环左移 k 位
```

把字节的 8 个比特位像拨盘一样旋转。

**加密：** `c = rotl8(p, k % 8)`
**解密：** `p = rotr8(c, k % 8)`

示例（k=3）：
```
原始:  10110010
旋转3: 10010101  ← 左边3位移到右边
```

### 4. NSwp —— 半字节交换 + 异或

```
密文 = swap(明文高低4位) XOR 密钥字节
```

先交换字节的高低半字节（nibble），再异或密钥。

**加密：** `c = nibble_swap(p) ^ k`
**解密：** `p = nibble_swap(c ^ k)`

示例：
```
原始:    1011 0010  (0xB2)
交换:    0010 1011  (0x2B)
XOR k:   0010 1011 ^ 0000 0011 = 0010 1000  (0x28)
```

## 轮转机制

```
字节位置:  0    1    2    3    4    5    6    7    ...
算法:     Add  XOR  Rot  NSwp Add  XOR  Rot  NSwp ...
密钥:     k[0] k[1] k[2] k[3] k[0] k[1] k[2] k[3] ...
```

4 种算法 × 4 字节密钥 = 每 4 字节一个完整周期。

## 为什么更安全？

| 攻击方式 | 单算法 | 多算法轮转 |
|----------|--------|------------|
| 已知明文攻击 | ❌ 一个字节破解全部 | ✅ 只能破解对应算法位置 |
| 频率分析 | ❌ 相同字符→相同模式 | ✅ 相同字符→不同密文 |
| 暴力破解 | ❌ 4字节密钥空间 | ✅ 4字节密钥 × 4种算法组合 |

## Nesh 增强模式

Nesh 模式使用 BLAKE2b 哈希函数进行密钥派生：

```
derived_key = BLAKE2b(key + data)[:4]
```

明文内容参与密钥派生，使得相同密钥加密不同明文时产生不同的派生密钥。

⚠️ 注意：Nesh 增强模式是**单向加密**，因为解密需要原始明文来派生密钥。
